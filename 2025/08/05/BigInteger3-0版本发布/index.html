BigInteger3.0版本发布

更新

在一年前，我编写了 BigInteger2 项目。BigInteger3.0 对 BigInteger2 进行了重构，主要更新是将动态分配空间改为 std::vector 自动分配空间，并将实现精细化。

操作文档

BigInteger 3.0 版本支持多种操作。下面的复杂度分析中，w=8,w′=4w=8, w'=4w=8,w′=4。

初始化

  • BigInteger()：创建一个新的 BigInteger，默认值为 000。
  • BigInteger(const BigInteger& x)：创建一个新的 BigInteger，值为 xxx，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为 xxx 的长度。
  • BigInteger(int64_t x)：创建一个新的 BigInteger，值为 xxx，时间复杂度 O(log⁡x)O(\log x)O(logx)。
  • BigInteger(const std::string& s)：从字符串创建一个新的 BigInteger，时间复杂度 O(n)O(n)O(n)，其中 nnn 为字符串长度。合法的字符串必须由若干个 - 号后接若干数字字符组成。
  • BigInteger(const std::vector<bool>& v)：从二进制表示创建一个新的 BigInteger，时间复杂度 O(n2)O(n^2)O(n2)，其中 nnn 为二进制表示长度。
  • BigInteger.from_int128(__int128 x)：static 型函数，从 __int128 类型创建一个新的 BigInteger，值为 xxx，时间复杂度 O(log⁡x)O(\log x)O(logx)。在不支持 __int128 的环境中，BigInteger 无此操作。

I / O

  • std::cin >> x：输入一个 BigInteger 的值，时间复杂度 O(n)O(n)O(n)，其中 nnn 为字符串长度。合法输入与从字符串初始化 BigInteger 的要求相同。
  • std::cout << x：输出一个 BigInteger 的值，时间复杂度 O(n)O(n)O(n)，其中 nnn 为此整数的长度。

类型转换

  • a.to_string()：返回值为 std::string 类型，返回 a 转换为字符串后的结果。时间复杂度 O(n)O(n)O(n)，其中 nnn 为此整数的长度。
  • a.to_int64()：返回值为 int64_t 类型，返回 a 转换为 64 位整数后的结果，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为此整数的长度。若结果发生溢出，则行为未定义。
  • a.to_binary()：返回值为 std::vector<bool> 类型，返回 aaa 的二进制表示，时间复杂度 O(n2)O(n^2)O(n2)，其中 nnn 为此整数的长度
  • a.to_int128()：返回值为 __int128 类型，返回 a 转换为 128 位整数后的结果，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为此整数的长度。若结果发生溢出，则行为未定义。在不支持 __int128 的环境中，BigInteger 无此操作。

基本运算

  • a.zero()：判断 aaa 是否为 000，时间复杂度 O(1)O(1)O(1)。

  • !a：判断 aaa 是否不为 000，时间复杂度 O(1)O(1)O(1)。

  • a.positive()：判断 aaa 是否为正数，时间复杂度 O(1)O(1)O(1)。000 不是正数。

  • a.negative()：判断 aaa 是否为负数，时间复杂度 O(1)O(1)O(1)。

  • a.compare(const BigInteger& b)：返回 aaa 与 bbb 比较的结果，返回值为 int 类型。若 a<ba<ba<b 返回 −1-1−1，a=ba=ba=b 返回 000，a>ba>ba>b 返回 111。时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为两整数的长度的较大值。

  • a <=> b, a <= b, a < b, a == b, a != b, a > b, a >= b：返回 aaa 与 bbb 比较的对应结果，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为两整数的长度的较大值。

  • -a：返回 −a-a−a，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为此整数的长度。

  • ~a：返回 −a−1-a-1−a−1，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为此整数的长度。

  • a.abs()：返回 ∣a∣|a|∣a∣，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为此整数的长度。

  • a + b：返回 a+ba+ba+b，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为两整数的长度的较大值。支持 a += b 原地加法。当 b 为 int32_t 类型时速度更快。

  • a - b：返回 a−ba-ba−b，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为两整数的长度的较大值。支持 a -= b 原地减法。当 b 为 int32_t 类型时速度更快。

  • a * b：返回 a×ba \times ba×b，时间复杂度 O(nlog⁡nw′)O(\dfrac{n \log n}{w'})O(w′nlogn​)，其中 nnn 为两整数的长度的较大值。当 n<n <n< 8 * FFT_LIMIT 时，采用 O(n2w2)O(\dfrac{n^2}{w^2})O(w2n2​) 竖式乘法计算。FFT_LIMIT 默认为 888。特别地，当 b 为 int32_t 类型时时间复杂度为 O(nw)O(\dfrac{n}{w})O(wn​)，且支持原地乘法。n>220n > 2^{20}n>220 时抛出 FFTLimitExceededError 异常。

  • a.square()：返回 a2a^2a2，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为此整数的长度，快于 a * a。

  • a.half()：返回 ⌊a2⌋\lfloor \dfrac{a}{2} \rfloor⌊2a​⌋，时间复杂度 O(nw)O(\dfrac{n}{w})O(wn​)，其中 nnn 为此整数的长度，快于 a / 2。

  • a / b：返回 ⌊ab⌋\lfloor \dfrac{a}{b} \rfloor⌊ba​⌋，时间复杂度 O(nlog⁡nw′)O(\dfrac{n \log n}{w'})O(w′nlogn​)，其中 nnn 为两整数的长度的较大值。当 n<n <n< 8 * NEWTON_DIV_LIMIT 时，采用 O(n2w)O(\dfrac{n^2}{w})O(wn2​) 试商除法计算。NEWTON_DIV_LIMIT 默认为 323232。特别地，当 b 为 int64_t 类型时时间复杂度为 O(nw)O(\dfrac{n}{w})O(wn​)，且支持原地除法。b=0b=0b=0 时抛出 ZeroDivisionError 异常。

  • a % b：返回 a mod ba \bmod bamodb，时间复杂度与 a / b 一致。b=0b=0b=0 时抛出 ZeroDivisionError 异常。

  • a.divmod(b)：返回一个 std::pair，分别为 (⌊ab⌋,a mod b)(\lfloor \dfrac{a}{b} \rfloor, a \bmod b)(⌊ba​⌋,amodb)，时间复杂度与 a / b 一致，但 bbb 为 int64_t 类型时无优化。b=0b=0b=0 时抛出 ZeroDivisionError 异常。

  • a.mod2()：返回 a mod 2a \bmod 2amod2，时间复杂度 O(1)O(1)O(1)。

  • a.pow(b)：返回 aba^bab，时间复杂度 O(nblog⁡nbw)O(\dfrac{nb \log nb}{w})O(wnblognb​)，其中 nnn 为此整数的长度。bbb 应为 int64_t 类型。

  • a.pow(b, p)：返回 ab mod pa^b \bmod pabmodp，时间复杂度 O(nblog⁡nbw)O(\dfrac{nb \log nb}{w})O(wnblognb​)，其中 nnn 为此整数的长度。bbb 应为 int64_t 类型，ppp 为 BigInteger 类型。

  • a.sqrt()：返回 ⌊a⌋\lfloor \sqrt{a} \rfloor⌊a​⌋，时间复杂度 O(nlog⁡nw)O(\dfrac{n \log n}{w})O(wnlogn​)，其中 nnn 为此整数的长度。a<0a<0a<0 时抛出 NegativeRadicandError 异常。

  • a.root(x)：返回 ⌊ax⌋\lfloor \sqrt[x]{a} \rfloor⌊xa​⌋，时间复杂度 O(nlog⁡nw)O(\dfrac{n \log n}{w})O(wnlogn​)，其中 nnn 为此整数的长度。x≤0x \le 0x≤0 时抛出 NegativeRadicandError 异常。2∣x2 \mid x2∣x 且 a<0a < 0a<0 时抛出 NegativeRadicandError 异常。

  • a.gcd(b)：返回 gcd⁡(a,b)\gcd(a,b)gcd(a,b)，时间复杂度 O(n2w)O(\dfrac{n^2}{w})O(wn2​)，其中 nnn 为两整数的长度的较大值。

  • a.lcm(b)：返回 lcm⁡(a,b)\operatorname{lcm}(a,b)lcm(a,b)，时间复杂度 O(n2w)O(\dfrac{n^2}{w})O(wn2​)，其中 nnn 为两整数的长度的较大值。

  • a << x：返回 a×2xa \times 2^xa×2x，时间复杂度 O((n+x)log⁡(n+x)w′)O(\dfrac{(n+x) \log(n+x)}{w'})O(w′(n+x)log(n+x)​)，其中 nnn 为此整数的长度。

  • a >> x：返回 ⌊a2x⌋\lfloor \dfrac{a}{2^x} \rfloor⌊2xa​⌋，时间复杂度 O((n−x)log⁡(n−x)w′)O(\dfrac{(n-x) \log(n-x)}{w'})O(w′(n−x)log(n−x)​)，其中 nnn 为此整数的长度。

  • a & b：返回 a,ba,ba,b 的按位与，时间复杂度 O(n2)O(n^2)O(n2)，其中 nnn 为两整数的长度的较大值。

  • a | b：返回 a,ba,ba,b 的按位或，时间复杂度 O(n2)O(n^2)O(n2)，其中 nnn 为两整数的长度的较大值。

  • a ^ b：返回 a,ba,ba,b 的按位异或，时间复杂度 O(n2)O(n^2)O(n2)，其中 nnn 为两整数的长度的较大值。

其他函数

  • a._move_l(x)：返回 ∣n×10wx∣|n \times 10^{wx}|∣n×10wx∣，时间复杂度 O(nw+x)O(\dfrac{n}{w}+x)O(wn​+x)，其中 nnn 为此整数的长度。

优缺点

优点：

  • 高度封装，几乎支持 int 类型所有操作，无需手写。
  • 速度快，大部分操作已优化到一个较优秀的复杂度。
  • 代码长度较短。

缺点：

  • 位运算速度较慢。
  • 不支持超大位数整数的乘除法。具体地，位数大于 220=10485762^{20}=1048576220=1048576 时抛出 FFTLimitExceededError 异常。
  • 偶尔出现的 bug。
